# 1. Analysis (RESEARCH Mode Population)

## 需求澄清/深入挖掘 (参考启动会议日志)
用户报告ATV-Bilibili-demo项目无法正常播放杜比视界内容，会降级为HDR格式播放。这影响了高端用户的观看体验，特别是在支持杜比视界的Apple TV设备上。

## 代码/系统调研 (AR提供架构分析)

### 关键发现：编解码器替换问题
通过深入分析`BilibiliLive/Component/Player/BilibiliVideoResourceLoaderDelegate.swift`文件，发现了问题的根本原因：

**问题位置：** `addVideoPlayBackInfo`方法（第89-97行）
```swift
if codecs == "dvh1.08.07" || codecs == "dvh1.08.03" {
    supplementCodesc = codecs + "/db4h"
    codecs = "hvc1.2.4.L153.b0"  // ❌ 错误：替换为普通HEVC
    videoRange = "PQ"
} else if codecs == "dvh1.08.06" {
    supplementCodesc = codecs + "/db1p"
    codecs = "hvc1.2.4.L150"     // ❌ 错误：替换为普通HEVC
    videoRange = "PQ"
}
```

**问题分析：**
1. 代码强制将杜比视界编解码器（dvh1.xx.xx）替换为普通HEVC编解码器（hvc1.xx.xx）
2. 虽然设置了SUPPLEMENTAL-CODECS，但主CODECS字段被替换，导致AVPlayer无法识别杜比视界
3. 这种替换破坏了杜比视界的元数据传递，导致降级为普通HDR播放

### 苹果官方标准验证
根据苹果官方文档（WWDC21 "Explore HLS variants in AVFoundation"）：
- 杜比视界的正确HLS格式应为：`CODECS="dvh1.05.06,ec-3"`
- 苹果HDR元数据规范明确要求杜比视界使用'dvh'编解码器标识符
- VIDEO-RANGE应设置为"PQ"

## 技术约束与挑战
1. **兼容性考虑：** 原代码可能是为了兼容某些不支持杜比视界的设备
2. **Apple TV硬件支持：** 需要确认目标Apple TV设备支持杜比视界播放
3. **B站API响应：** 需要验证B站API返回的杜比视界流是否符合标准

## 隐含假设
1. 用户使用的Apple TV设备支持杜比视界播放
2. B站提供的杜比视界流格式符合苹果标准
3. 当前的编解码器替换是为了解决某种兼容性问题

## 早期边界情况考量
1. **设备兼容性：** 不支持杜比视界的设备如何处理
2. **网络条件：** 杜比视界流通常码率较高，网络条件不佳时的降级策略
3. **用户设置：** 用户可能关闭了杜比视界播放选项

## 初步风险评估
1. **低风险：** 修复编解码器替换逻辑，保持原始杜比视界标识符
2. **中风险：** 可能影响不支持杜比视界设备的兼容性
3. **高风险：** 如果B站API格式有问题，可能导致播放失败

## 知识空白

2. 不同Apple TV设备型号的杜比视界支持情况
3. B站杜比视界流的具体格式和质量参数

## 扩展兼容性问题分析 (新发现 - 2025-06-08 18:38:03)

### 问题升级：广泛的格式兼容性缺失
用户反馈表明修复后仍存在大量HDR和杜比视界格式无法识别：
1. **杜比视界格式变体缺失：** 当前只支持dvh1.08.07, dvh1.08.03, dvh1.08.06，但存在更多profile变体
2. **HDR10+格式未处理：** 没有专门的HDR10+格式识别和处理逻辑  
3. **降级机制过于激进：** HDR→SDR、杜比视界→HDR/SDR的不当降级
4. **编解码器解析不全面：** 可能存在格式检测逻辑的盲区

### 代码分析发现的实际问题
通过深入代码审查发现：

**核心问题：**

2. 缺乏系统性的格式识别框架  
3. 没有HDR10+的专门处理
4. 未知格式直接警告而非智能处理

### 苹果标准要求的完整格式支持
根据最新苹果文档和行业标准：

**杜比视界标准格式：**
- Profile 4: dvh1.04.xx (单层，基于BT.2020)
- Profile 5: dvh1.05.xx (双层，基于BT.2020) 
- Profile 7: dvh1.07.xx (单层，基于HDR10)
- Profile 8: dvh1.08.xx (双层，基于HDR10)

**Level支持范围：**
- Level 01-06: 对应不同分辨率和帧率组合
- 常见组合：05.05, 05.06, 07.06, 08.01, 08.03, 08.06, 08.07等

**HDR10+格式：**
- 可能使用hvc1/hev1 + 特殊元数据标识
- 需要专门的SUPPLEMENTAL-CODECS处理
