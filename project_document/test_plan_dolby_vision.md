## 1. 测试目标与范围

### 1.1 主要测试目标
- **验证杜比视界编解码器修复效果**：确保dvh1编解码器不被错误替换为hvc1
- **确保杜比视界内容正确播放**：验证不同Profile的杜比视界内容能够正确识别和播放
- **回归测试**：确保修复不影响现有HDR播放功能
- **兼容性验证**：确保在不同Apple TV设备上的播放兼容性

### 1.2 测试范围
- **编解码器处理逻辑**：BilibiliVideoResourceLoaderDelegate.swift中的addVideoPlayBackInfo方法
- **HLS内容生成**：验证生成的HLS内容包含正确的杜比视界元数据
- **播放功能**：端到端的杜比视界内容播放验证
- **设备兼容性**：Apple TV 4K (2021及以后)设备的播放测试

## 2. 测试策略

### 2.1 测试金字塔结构
```
    /\     端到端测试 (E2E)
   /  \    - 真实设备播放验证
  /____\   - 用户体验测试
 /      \  
/________\  集成测试 (Integration)
           - HLS生成验证
           - 播放流程测试
___________
单元测试 (Unit)
- 编解码器处理逻辑
- 方法输入输出验证
```

### 2.2 测试原则
- **AAA模式**：Arrange（准备）→ Act（执行）→ Assert（断言）
- **测试隔离**：每个测试用例独立，不依赖其他测试
- **可重复性**：测试结果在相同条件下可重复
- **全面覆盖**：覆盖所有杜比视界Profile和边界情况
